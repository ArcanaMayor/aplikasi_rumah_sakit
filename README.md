# Rumah Sakit App

Rumah Sakit App merupakan sebuah aplikasi berbasis web yang dikembangkan untuk membantu proses pengelolaan data dan administrasi rumah sakit secara digital. Aplikasi ini dibuat sebagai sarana pembelajaran dalam pengembangan perangkat lunak dan masih berada dalam tahap pengembangan.

Perlu ditegaskan bahwa aplikasi ini **masih dalam pengembangan** dan belum ditujukan untuk penggunaan secara nyata di lingkungan rumah sakit. Fokus utama pengembangan aplikasi ini adalah pada penerapan konsep pemrograman, pengelolaan database, serta pemahaman alur kerja sistem berbasis web.

---

## Status Pengembangan
Aplikasi ini **masih dalam tahap pengembangan (development stage)**. Beberapa fitur telah berjalan sesuai dengan perancangan awal, namun masih terdapat bagian yang memerlukan penyempurnaan, pengujian lanjutan, serta optimalisasi baik dari sisi performa maupun keamanan.

Pengembangan dilakukan secara bertahap dan menyesuaikan dengan kebutuhan pembelajaran serta kemampuan teknis pengembang pada saat proses pembuatan aplikasi berlangsung.

---

## Latar Belakang
Pengelolaan data rumah sakit merupakan aspek penting dalam sistem pelayanan kesehatan. Proses administrasi seperti pencatatan data pasien, data dokter, dan layanan pemeriksaan memerlukan sistem yang terstruktur agar data dapat dikelola dengan baik.

Melalui project ini, dibuat sebuah aplikasi sederhana sebagai simulasi sistem informasi rumah sakit. Aplikasi ini masih dalam pengembangan dan dirancang untuk memahami bagaimana sebuah sistem informasi dibangun dari tahap perancangan hingga implementasi.

Selain itu, project ini juga bertujuan untuk melatih kebiasaan dokumentasi dan penggunaan GitHub sebagai media penyimpanan serta pengelolaan source code.

---

## Tujuan Pengembangan
Tujuan utama pengembangan Rumah Sakit App antara lain:
1. Melatih kemampuan backend development
2. Menerapkan konsep CRUD dalam aplikasi nyata
3. Memahami struktur database relasional
4. Mengimplementasikan autentikasi pengguna
5. Membiasakan penggunaan Git dan GitHub
6. Menyusun dokumentasi project secara terstruktur
7. Mengembangkan aplikasi secara bertahap (iterative development)

Seluruh tujuan tersebut masih terus dikembangkan seiring berjalannya proses pembuatan aplikasi.

## Ruang Lingkup Sistem
Ruang lingkup aplikasi ini masih terbatas pada pengelolaan data administrasi dasar rumah sakit. Sistem ini **belum mencakup proses medis secara detail** dan masih berada dalam tahap pengembangan awal.

Cakupan sistem meliputi:
- Pengelolaan data pengguna
- Pengelolaan data pasien
- Pengelolaan data dokter
- Pencatatan layanan pemeriksaan
- Tampilan dashboard admin

Pengembangan lanjutan direncanakan untuk memperluas ruang lingkup sistem.

## Fitur Aplikasi (Sementara)

### Autentikasi Pengguna
Sistem login digunakan untuk membatasi akses pengguna. Fitur ini masih dalam pengembangan dan akan terus disempurnakan dari sisi keamanan.

### Manajemen Data Pasien
Pengguna dapat menambahkan, melihat, mengubah, dan menghapus data pasien. Fitur ini masih dapat dikembangkan lebih lanjut, terutama dalam validasi data.

### Manajemen Data Dokter
Fitur ini digunakan untuk mengelola data dokter yang terdaftar dalam sistem. Saat ini masih dalam pengembangan dan terbuka untuk penambahan atribut data.

### Pencatatan Layanan
Pencatatan layanan atau pemeriksaan pasien masih bersifat sederhana dan akan dikembangkan lebih lanjut.

### Dashboard
Dashboard menampilkan ringkasan data yang ada di dalam sistem. Tampilan dan fungsinya masih dalam tahap pengembangan.
